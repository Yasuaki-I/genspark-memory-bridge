## 判明したこと

- turndown は Genspark の AI応答 HTML 構造に対して問題なく動作する。自前変換器へのフォールバックは不要
- Genspark UI の Copy ボタンが `<pre>` 内に「Copy」テキストを埋め込んでいる → T1-3で前処理（pre内のコピーボタン要素を除去）が必要
- DevTools コンソールで const 宣言した変数はコンテキストリフレッシュで失われる → 検証中は window.* に格納する方が安全
- Genspark のページ上で外部スクリプトの動的ロードが許可されている（CSP制約なし）
- Genspark チャット命名規則は「{プロジェクト名}_{連番3桁}」で固定（プロジェクト名はキャメルケース、連番ゼロパディング）
- 出力形式の合意: Obsidian保存用ドキュメントは応答全体を1つのコードブロックで囲む。差分更新時は変更箇所のみ出力する
