## プロジェクト概要

Chrome 拡張で Genspark チャット全文を Markdown 化し GitHub に保存、次回チャット開始時に自動検知して復元（MVP: クリップボードへコピー）。Obsidian と GitHub 同期。

## ゴール

1. **保存フロー**: チャット終了 → 拡張ボタン → Markdown 化 → GitHub commit
2. **復元フロー**: 新チャット開始 → 拡張がチャット名パターン検知 → GitHub からログ＋引き継ぎドキュメント取得 → クリップボードへコピー → 手動 Ctrl+V ペースト（MVP）
3. **ドキュメント統合**: Obsidian ↔ GitHub 同期

## 競合調査結果まとめ

調査の結果、GMBと関連する領域には**3つのカテゴリ**の既存ツールが存在しています。

* * *

**カテゴリ1: エクスポート特化型（保存のみ）**

「Genspark to Obsidian」（Chrome Web Store公開済み・v1.2.0）が**直接競合**です。GensparkチャットをバッチでMarkdown化してObsidianに保存する機能を持っています。YAML frontmatter付き、Mermaid図やコードブロック保存対応、プライバシーファースト（ローカル処理）。つまりGMBのPhase 1「保存フロー」とほぼ同じ機能を既に提供しています。

* * *

**カテゴリ2: クロスプラットフォーム文脈転送型**

「LLM Context Bridge」と「Context Sync」が該当します。AI間の会話コンテキストをワンクリックで別のAIに転送する拡張です。Context Syncは特に高機能で、Claude/ChatGPT/Gemini/DeepSeekのDOM自動スクレイピング、自動注入（テキストエリアへの自動ペースト＋自動送信）、JSON形式のカプセル化まで実現しています。

* * *

**カテゴリ3: AIメモリレイヤー型（長期記憶）**

「OpenMemory（Mem0）」「MemoryPlugin」「AI Context Flow」「MaxMemory」などが該当します。会話から重要事実を自動抽出し、構造化メモリとして保持、次回以降のプロンプトに自動注入するアプローチです。OpenMemoryはChatGPT/Claude/Gemini/Perplexity/Grok対応で、完全無料・ローカル動作。

* * *

## GMBの現在地と差別化の論点

率直に言うと、**GMBのPhase 1〜2の機能は既に市場にある複数ツールでカバーされています。** ただし、GMBには以下の独自性があります：

-   **GitHub連携による版管理**：保存先がGitHubであり、Obsidian-Git連動によるバージョン管理が自然にできる
-   **「同じチャットの続き」を検知して復元する**：他ツールが「AI間の乗り換え」に注力する中、GMBは「同一プラットフォーム内でのセッション継承」に特化

# Genspark Memory Bridge - プライバシーポリシー

最終更新日: 2026-06-06

## はじめに

Genspark Memory Bridge（以下「本拡張」）は、ユーザーのプライバシーを
尊重し、最小限のデータのみを取り扱います。本ポリシーでは、本拡張が
どのようなデータを扱い、どのように保護しているかを説明します。

## 収集するデータ

本拡張は以下のデータを取り扱います。

- Gensparkチャットページ上のメッセージ内容（Markdown変換のため）
- ユーザーが設定画面に入力するGitHub認証情報（ユーザー名、リポジトリ名、
  Personal Access Token）

## データの保存場所

- GitHub認証情報: ユーザーのブラウザ内（chrome.storage.local）にのみ保存
  されます。外部サーバーへの送信は一切行いません。
- チャット内容: ユーザーが「保存」操作を実行した場合にのみ、ユーザー自身の
  GitHubリポジトリへ送信されます。本拡張の開発者が管理するサーバーには
  一切送信されません。

## 外部との通信

本拡張が通信する外部サービスはGitHub API（api.github.com）のみです。
通信はすべてHTTPSで暗号化されます。通信が発生するタイミングは以下に
限定されます。

- ユーザーが「保存」ボタンを押した時（チャット内容のcommit）
- ユーザーが「復元」操作を行った時（引き継ぎドキュメントの取得）
- ユーザーが保存済みチャット一覧を表示した時（ファイル一覧の取得）

ユーザーの操作なしに自動で外部通信を行うことはありません。

## 第三者へのデータ提供

本拡張は、ユーザーのデータを第三者に販売、提供、共有することは
一切ありません。

## アナリティクス・トラッキング

本拡張は、Google Analytics、広告SDK、その他のトラッキングツールを
一切使用しません。ユーザーの行動を追跡・記録する機能は含まれていません。

## データの削除

ユーザーはいつでも以下の方法でデータを削除できます。

- chrome.storage.local のデータ: 拡張機能をアンインストールすることで
  自動的に削除されます。
- GitHubリポジトリ上のデータ: ユーザー自身のリポジトリ内にあるため、
  通常のGit操作で自由に削除できます。

## 権限の使用目的

| 権限 | 使用目的 |
|------|----------|
| activeTab | Gensparkチャットページのメッセージ内容を読み取るため |
| storage | GitHub認証情報と拡張設定をブラウザ内に保存するため |
| tabs | アクティブタブのURL判定（Gensparkページかどうかの確認）に使用 |
| host_permissions (github.com) | GitHub APIへのファイル保存・取得リクエストのため |
| host_permissions (genspark.ai) | Gensparkページ上でcontent_scriptを実行するため |

## 変更について

本ポリシーに重要な変更がある場合、拡張のアップデートノートにて
通知します。

## お問い合わせ

本ポリシーに関するご質問は、GitHubリポジトリのIssuesにてお受けします。

Privacy Policy - Genspark Memory Bridge

function mapGitHubError(response, errorBody) {
  const status = response.status;
  const original = (errorBody && errorBody.message) ? errorBody.message : "";

  if (status === 401) {
    return "PAT（Personal Access Token）が無効か期限切れです。GitHubで新しいPATを発行し、設定を更新してください。";
  }

  if (status === 403) {
    const remaining = response.headers.get("X-RateLimit-Remaining");
    if (remaining === "0") {
      return "GitHub APIのレート制限に達しました。しばらく待ってから再試行してください（通常1時間で復旧）。";
    }
    return "PATにリポジトリへの書き込み権限がありません。GitHub設定でPATの 'repo' スコープを有効にしてください。";
  }

  if (status === 404) {
    return "リポジトリが見つかりません。ユーザー名・リポジトリ名の綴りと、PATがそのリポジトリにアクセス可能か確認してください。";

}

async function saveToGitHub(token, repo, path, content, commitMessage) {
  const url = `https://api.github.com/repos/${repo}/contents/${path}`;

  let sha = null;
  try {
    const checkResponse = await fetch(url, {
      headers: {
        Authorization: `Bearer ${token}`,
        Accept: "application/vnd.github.v3+json",
      },
    });
    if (checkResponse.ok) {
      const existing = await checkResponse.json();
      sha = existing.sha;
    }
  } catch (e) {
    // ネットワークエラー等。新規作成PUT側で再度エラーが出るのでここでは続行
  }

  const body = {
    message: commitMessage,
    content: btoa(unescape(encodeURIComponent(content))),
  };
  if (sha) {
    body.sha = sha;
  }

  const response = await fetch(url, {

      Accept: "application/vnd.github.v3+json",
      "Content-Type": "application/json",

    body: JSON.stringify(body),
  });

  if (!response.ok) {
    const errorBody = await safeJson(response);
    throw new Error(mapGitHubError(response, errorBody));
  }

  return await response.json();
}
