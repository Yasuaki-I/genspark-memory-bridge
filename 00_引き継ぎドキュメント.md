## プロジェクト概要

GensparkのAIチャットにメモリ永続化の仕組みを構築するプロジェクト。Chrome拡張機能でチャット内容のGitHub保存・新チャットへの復元を実現する。

### ゴール（完成状態の定義）

1. 保存フロー: Gensparkチャット終了 → Chrome拡張のボタン押下 → 会話全文がMarkdown化されGitHub Repoにcommit
2. 復元フロー: 新チャット開始 → Chrome拡張がチャット名パターン（OpenClaw_00x等）を検知 → GitHubから前回ログ＋引き継ぎドキュメントを取得 → クリップボードにコピー → 手動ペースト（MVP）
3. ドキュメント統合: ObsidianドキュメントがGitHub経由で同期されている

## 技術スタック

- Chrome拡張機能（Manifest V3）
- GitHub API
- Obsidian + Obsidian Git
- turndown v7.2.0（lib/turndown.js として同梱）

## 通信仕様メモ（5/10時点）

content_script.js が応答するアクション:

- `extractChat`（action キー）→ {success, markdown, messageCount, meta}
- `previewFilename`（action キー）→ {ok, filename, filePath, chatTitle, messageCount}
- `buildMarkdown`（action キー）→ {ok, data}
- `GMB_BUILD_MARKDOWN`（type キー・background互換エイリアス）→ {ok, data}

background.js は `chrome.tabs.sendMessage(tabId, {type:"GMB_BUILD_MARKDOWN"})` で問い合わせ、`{ok:true, data:{markdown, meta}}` を期待する。

background.js のエラーマッピング:

- `mapGitHubError(response, errorBody)`: HTTPステータスを日本語メッセージに変換
- `mapNetworkError(error)`: TypeError（fetch失敗）を「ネットワーク接続を確認してください」に変換、それ以外は元メッセージを返す
- `safeJson(response)`: response.json() を安全に取得、失敗時は空オブジェクト
- `checkSettings(settings)`: PAT/repo/user の欠落を識別し具体的な未設定項目を含むメッセージを返す

- Obsidian保存ドキュメントは応答全体を1つのコードブロックで囲む
- 内側コードブロックは ~~~（チルダフェンス）

- 拡張機能ソースコード（content_script.js等）はObsidian管理外
- ソース修正時は全文出力

### 装飾注釈の禁止【5/9新設・最重要】

- AIアシスタントは、コードブロック内に図解用の装飾注釈（例: `← ★ここを残す`）を絶対に入れない

- ユーザーがコードブロック全体をコピペしても安全な状態を保証する

### バックアップ運用ルール【5/10改訂・選択肢C採用】

手元バックアップの保持方針:

直近タスクの完了時点バックアップを2世代まで保持する。具体的には「現在着手中タスクの直前タスク完了版」+「1つ前のタスク完了版」の合計2世代を手元に保持する。これより古いバックアップは削除し、必要時は GitHub のコミット履歴から取得する。

理由:

ロールバック先として現実的なのは直近1〜2タスク前まで。手元バックアップは即時復元用、GitHub は長期履歴用、と役割分担する。手元バックアップがあればネット切断時・GitHub障害時・git操作不慣れな状況下でも、ファイルを上書きコピーするだけで即座に復元できる。

命名規則:

`{ファイル名}.bak_{YYYYMMDD}_{次タスクID}_pre`

例: `background.js.bak_20260510_t18_pre` は T1-7 完成時点（T1-8 着手直前）のバックアップ。日付は次タスク着手日、タスクIDは次タスクのID。

整理タイミング:

各タスク完了時に、新しいバックアップを取得した直後に「2世代前」を削除する。整理コマンドは引き継ぎドキュメントに記載せず、毎回チャットで案内する（非エンジニア配慮）。

Phase 完了時:

Phase 1 完了判定（T1-10）後に、Phase 1 全体の最終版バックアップ1セットを `bak_phase1_final` として保持し、それ以外を削除。Phase 2 開始時のロールバック先を確保する。

GitHub からの過去版復元手順:

1. https://github.com/Yasuaki-I/genspark-memory-bridge/commits/main を開く
2. 復元したい時点のコミットをクリック
3. 該当ファイルの「View file」または「Raw」ボタンをクリック
4. ブラウザで「名前を付けて保存」→ ローカルの作業フォルダに保存

## DevTools操作の注意点

- Content ScriptのIsolated Worldのため、DevTools Consoleコンテキストを「Genspark Memory Bridge」に切り替える必要あり
- F5後はコンテキストが「top」に戻るので毎回切替
- AdBlockの ERR_BLOCKED_BY_CLIENT は無視
- chrome://extensions のエラーバッジは過去ログの蓄積で残ることがある。「すべて削除」で消してから動作確認するとクリーンな判定ができる
- content_script.js 110行目の `console.debug` は意図的にエラーバッジに出さない設計（5/10改修）。万一エラーバッジが出る場合は別の問題
