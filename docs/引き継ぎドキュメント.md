## 第1部 サマリ・運用基盤

### 1.1 このドキュメントの使い方

新しいチャットを開始した際、本ファイルの内容を全文貼り付けることで
引き継ぎが完了する。状態が変わったとき・作業の区切り・チャット終了時に更新する。

### 1.2 このチャットの役割

GensparkMemoryBridge Chrome拡張の設計・実装・検証・販売準備。
Phase 1・Phase 2の技術実装、T-VAL検証の実施、Claude Projects連携設計、
Chrome Web Store公開準備、note第3弾（GMB設計判断）の素材整理。
Claude-Chat（全体マネジメント）とは独立して、GMB開発固有の技術判断を管理。

### 1.3 引き継ぎ品質を保つための3原則

- **「なぜ」を残す**: 判断の結論だけでなく、その判断に至った理由・前提・
  代替案の比較を記録する。後続チャットが同じ判断軸で議論を継続できるようにする。
- **失敗を構造化する**: 失敗事象は「症状→仮説→検証→真因→対処→再発防止」
  の構造で記録する。教訓を抽象化して再利用可能な知見にする。
- **状態と方針を分離する**: 現在の状態（事実・数値・進捗）と、方針
  （判断・優先順位・戦略）を明確に分けて記述する。状態は更新頻度が高く、
  方針は安定的。

- ADPは「売り先」ではなく「無料テスト場＆事例提供者」

### 4.2 競合調査結果（2026-06-06実施）

3カテゴリに分類:

カテゴリ1 エクスポート特化型（保存のみ）:
- 「Genspark to Obsidian」（Chrome Web Store公開済・無料・v1.2.0）
- GMB Phase 1とほぼ同等機能。復元フローなし。

カテゴリ2 クロスプラットフォーム文脈転送型:
- 「Context Sync」: Claude/ChatGPT/Gemini/DeepSeek対応、自動注入＋自動送信
- 「LLM Context Bridge」: AI間ワンクリック転送
- GMBとは方向性が異なる（AI間乗り換え vs 同一プラットフォーム内セッション継承）

カテゴリ3 AIメモリレイヤー型:
- 「OpenMemory（Mem0）」: 重要事実自動抽出・プロンプト自動注入・無料
- 「MemoryPlugin」「AI Context Flow」「MaxMemory」

GMBの差別化ポイント:
- 復元フロー付きGenspark特化ツールは確認範囲内で唯一
- 「引き継ぎドキュメント（コンテキスト）」による構造化された状態継承
- GitHub連携による版管理（Obsidian-Git連動）
- 同一プラットフォーム内でのセッション継承に特化
