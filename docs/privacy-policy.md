# Genspark Memory Bridge - プライバシーポリシー

最終更新日: 2026-06-06

## はじめに

Genspark Memory Bridge（以下「本拡張」）は、ユーザーのプライバシーを
尊重し、最小限のデータのみを取り扱います。本ポリシーでは、本拡張が
どのようなデータを扱い、どのように保護しているかを説明します。

## 収集するデータ

本拡張は以下のデータを取り扱います。

- Gensparkチャットページ上のメッセージ内容（Markdown変換のため）
- ユーザーが設定画面に入力するGitHub認証情報（ユーザー名、リポジトリ名、
  Personal Access Token）

## データの保存場所

- GitHub認証情報: ユーザーのブラウザ内（chrome.storage.local）にのみ保存
  されます。外部サーバーへの送信は一切行いません。
- チャット内容: ユーザーが「保存」操作を実行した場合にのみ、ユーザー自身の
  GitHubリポジトリへ送信されます。本拡張の開発者が管理するサーバーには
  一切送信されません。

## 外部との通信

本拡張が通信する外部サービスはGitHub API（api.github.com）のみです。
通信はすべてHTTPSで暗号化されます。通信が発生するタイミングは以下に
限定されます。

- ユーザーが「保存」ボタンを押した時（チャット内容のcommit）
- ユーザーが「復元」操作を行った時（引き継ぎドキュメントの取得）
- ユーザーが保存済みチャット一覧を表示した時（ファイル一覧の取得）

ユーザーの操作なしに自動で外部通信を行うことはありません。

## 第三者へのデータ提供

本拡張は、ユーザーのデータを第三者に販売、提供、共有することは
一切ありません。

## アナリティクス・トラッキング

本拡張は、Google Analytics、広告SDK、その他のトラッキングツールを
一切使用しません。ユーザーの行動を追跡・記録する機能は含まれていません。

## データの削除

ユーザーはいつでも以下の方法でデータを削除できます。

- chrome.storage.local のデータ: 拡張機能をアンインストールすることで
  自動的に削除されます。
- GitHubリポジトリ上のデータ: ユーザー自身のリポジトリ内にあるため、
  通常のGit操作で自由に削除できます。

## 権限の使用目的

| 権限 | 使用目的 |
|------|----------|
| storage | GitHub認証情報と拡張設定をブラウザ内に保存するため |
| clipboardWrite | 復元したコンテキストをクリップボードにコピーするため |
| activeTab | Gensparkチャットページのメッセージ内容を読み取るため |
| host_permissions (genspark.ai) | Gensparkページ上でcontent_scriptを実行し、チャット内容を取得するため |
| host_permissions (api.github.com) | GitHub APIへのファイル保存・取得リクエストのため |

## 変更について

本ポリシーに重要な変更がある場合、拡張のアップデートノートにて
通知します。
